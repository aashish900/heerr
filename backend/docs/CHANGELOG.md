## 2026-06-08 ‚Äî D1 jobs state-machine service

- `backend/app/services/jobs.py` ‚Äî pure async functions taking an `AsyncSession`. Service does **not** commit; callers (the FastAPI endpoint via `get_session`) commit.
  - `create_job_idempotent(session, *, spotify_uri, spotify_type, token_id) -> tuple[Job, bool]` ‚Äî checks for an existing active job; if absent, INSERTs a new `queued` row via `async with session.begin_nested(): await session.flush()` so an `IntegrityError` (caused by the partial unique index `jobs_active_uri_idx` losing a race with a concurrent insert) only rolls back the SAVEPOINT, not the outer transaction. On `IntegrityError`, re-fetches and returns the winner with `deduped=True`.
  - `mark_running(job_id)` ‚Äî UPDATE ‚Ä¶ WHERE state='queued'; raises `InvalidStateTransition` on `rowcount=0`. Sets `started_at=now()`.
  - `mark_done(job_id)` ‚Äî UPDATE ‚Ä¶ WHERE state='running'; sets `finished_at=now()`.
  - `mark_failed(job_id, error_msg)` ‚Äî UPDATE ‚Ä¶ WHERE state IN ('queued','running'); sets `finished_at`, `error_msg`.
  - `bump_attempt(job_id)` ‚Äî `attempt_count = attempt_count + 1` (no transition check; retry path uses it).
  - `find_active_for_uri(spotify_uri) -> Job | None` ‚Äî partial-index-backed lookup.
  - `find_download_for_track(spotify_track_uri) -> Download | None`.
- `InvalidStateTransition` exception class for illegal transitions.
- `backend/tests/test_jobs_service.py` ‚Äî 16 tests:
  - create: fresh insert (`deduped=False`); active-exists dedupe; re-queue allowed after `done`; **concurrent inserts via `asyncio.gather`** end with one row and exactly one `deduped=True` ‚Äî proves the DB-level race protection.
  - `mark_running` from queued ‚úì; rejects already-running.
  - `mark_done` from running ‚úì; rejects from queued.
  - `mark_failed` from queued ‚úì; from running ‚úì; rejects from done.
  - `bump_attempt` increments by N over N calls.
  - Finders: active job found; cleared after `done`; download lookup hit + miss.
